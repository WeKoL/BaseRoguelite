# BaseRoguelite v0.4.0 — логическое ядро

Эта версия делает упор не на новые сцены Godot, а на самостоятельный слой игровой логики в `Core/Version040Logic`.

Главная цель: получить ядро, которое можно тестировать отдельно от визуальной части и постепенно подключать к сценам, меню и объектам мира.

## Что добавлено

- единый реестр предметов `ItemRegistry040`;
- чистый инвентарь `InventoryCore040` со стаками, переносом, разделением и сортировкой;
- экипировка `EquipmentCore040` с безопасной заменой предметов;
- хранилище `StorageCore040` с лимитами, резервами и аудитом;
- авторазгрузка по категориям предметов;
- рецепты, книга рецептов, проверка станков и крафт из хранилища;
- очередь крафта;
- состояние игрока: HP, стамина, еда, вода, статусы;
- расходники через `ConsumableUseCore040`;
- ресурсные узлы, инструменты, добыча и loot table;
- бой, ИИ врага, волны врагов, патроны;
- мир, зоны, события экспедиции, эвакуация на базу;
- база: уровни построек, энергия, оборона, комфорт, улучшения, рейды;
- квесты: цели, прогресс, награды;
- сохранения: snapshot, JSON, миграция, валидация;
- UI view-model: HUD, меню действий, tooltip;
- feedback bus и планировщик feedback-событий;
- quality gate и баланс-валидатор;
- интеграционный `GameCore040` для тестового игрового цикла без Godot-сцен.

## Где тесты

`Tests/BaseRoguelite.Tests/Version040/Version040CoreTests.cs`

Там 15 тестов: по одному на каждую из 14 веток и один интеграционный тест полного core-loop.

## Как использовать дальше

1. Сначала добиться успешного `dotnet test`.
2. Потом подключать `GameCore040` или отдельные core-сервисы в Godot-сцены.
3. Не тащить UI-логику прямо в `Core`: Godot должен только отображать состояние и вызывать методы ядра.
